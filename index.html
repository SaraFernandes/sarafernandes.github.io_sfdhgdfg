Minha Missão

Tenho como missão acompanhar ativamente todo o ciclo de vida de desenvolvimento de um software, de forma a, juntamente com o cliente, avaliar requisitos, alinhar expectativas e gerar bons resultados, além de lidar com possíveis imprevistos.

Analista de Requisitos

Levantamento de requisitos para desenvolvimento ou implementação de sistemas e também de demandas de melhorias relacionadas ao software; relatórios (documento de visão, documento de especificação funcional de requisitos, casos de uso, classes, regras de negócios) segundo UML; modelagem de banco de dados, elaboração de manuais de usuário (de sistemas) e prototipação de telas.

Analista de Testes de Software

Elaboração de condições e casos de testes; documentação de casos de testes; execução de testes (usuário, regressão, aceitação, caixa-preta, teste de desempenho / carga / estresse); report aos gerentes e desenvolvedores e acompanhamento das homologações junto aos clientes.

  • A Engenharia/Análise de Requisitos

    É uma sub-área da Engenharia de Software, estuda o processo de definição dos requisitos que o software deve atender. A área surgiu em 1993 com a realização do I International Symposium on Requirements Engineering. O processo de definição de requisitos é uma interface entre os desejos e necessidades dos clientes e a posterior implementação desses requisitos em forma de software.

  • Teste de Software

    É o processo que consiste em todas as atividades do ciclo de vida, tanto estáticas quanto dinâmicas, voltadas para o planejamento, preparação e avaliação de produtos de software e produtos de trabalho relacionados a fim de determinar se eles satisfazem os requisitos especificados e demonstar que estão aptas para sua finalidade e para a detecção de defeitos. (Glossário de Termos do ISTQB)
